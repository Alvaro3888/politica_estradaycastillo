Resumen

En el sistema se valora la privacidad y seguridad de la información de los clientes, proveedores, vendedores, etc. Se va a explicar cómo se recopila, utiliza y protege la información ingresada en las diferentes ventanas del sistema. También se mostrará como circula la Información en el sistema a través de un Diagrama de Flujo de Datos (DFD) y como se interactúa con el Sistema en el Diagrama de Contexto (DC).

Se mostrará el Diseño de Interfaces, para que se tenga una idea mejor de cómo están conectadas las ventanas, y las diferentes acciones que se podrán llevar a cabo en ellas detalladas posteriormente

Se mostrará un Manual en el que podrá ver los pasos que el Usuario va a seguir, para su correcto funcionamiento.



3

Introducción

El sistema de Gestión de Proveedores, Clientes, Productos y Ventas es una herramienta diseñada para ayudar al Cliente a gestionar de manera eficiente las relaciones los con proveedores, clientes y productos, así como las Ventas. Este sistema busca automatizar y simplificar las tareas del Local, permitiendo al Cliente centrarse en el crecimiento y desarrollo de dicho Local.

Con este sistema, se podrá gestionar de manera efectiva sus proveedores, clientes y productos, así como los pedidos, facturas y pagos. Este simplifica la Administración y visualización de los Clientes y Proveedores. Y realiza un calculo Automatizado para que el Cliente, pueda ver sus Ganancias en tiempo Real de su Negocio.

Alcance

Esta política esta aplicada a proteger los datos de:

- Clientes y proveedores que sean ingresados en el sistema.

- Vendedores y empleados que sean ingresados en el

- Información recopilada a través de nuestro sistema.

5

Uso de la Información

Utilizamos la información antes mencionada para:

- Proporcionar productos a nuestros clientes.

- Gestionar relaciones con proveedores.

- Realizar análisis y reportes estadísticos.

- Mejorar la experiencia del usuario en nuestro sistema.

- Agilizar las ventas y el control.

Protección de la Información

- El sistema las secciones necesarias realiza un cifrado de determinados Datos (Como en el caso de la Contraseña)

- Restringimos el Acceso a los datos privados (Clientes, Proveedores) solo a los vendedores que utilicen el Sistema

Cambios a esta Política

- Los cambios realizados en la Política de Seguridad, serán publicados en el Sistema, para informarles a los usuarios la nueva actualización, y como les afecta.

Objetivos y Análisis

Descripción del Problema:

Enfrenta dificultades para gestionar de manera eficiente sus relaciones con (proveedores, clientes y ventas), es realizado actualmente en papel de talonarios y cuaderno de registros

Aunque esta herramienta permite cierta organización, su uso en este contexto presenta múltiples inconvenientes como:

• Retrasos: Realizar múltiples registros en formato papel es un proceso lento.

• Pérdida de datos: hay mas probabilidad de que se pierda el registro en formato papel a que se pierda del Sistema

• Trabajo excesivo: El control de Stock de productos puede ser muy tedioso y repetitivo

Objetivo General:

El objetivo general es realizar un sistema integral y eficiente para administrar las relaciones del Local (proveedores y clientes), y visualizar de manera más fácil e interactiva las ventas generadas

Todo esto con el fin de mejorar la productividad, reducir errores y aumentar la satisfacción del cliente.

Objetivos específicos:

• Automatizar la gestión de proveedores y clientes.

• Automatizar de manera eficiente las ventas.

• Facilitar el control de productos.

Beneficiarios:
